Projet n°2 · S4Mnémosyne — amélioration (équipe SATURNE 2)
ObjectifsTâches réaliséesCompétencesPreuvesPistes (BUT3)
Principal : reprendre une application Mnémosyne du S3 pour refondre sa base de données et enrichir le diagramme de Sankey (parcours plus lisibles).

Secondaire : moderniser l'interface (charte, page d'accueil allégée).
  • Analyse de l'existant : comparaison des 4 projets S3 sur 4 critères (langage, BDD, visuel, reprise), notés sur 20.
  • Choix argumenté de la base : reprise de notre projet Flask, en intégrant la modélisation d'un autre groupe et l'idée d'une table SCENARIO.
  • Refonte du modèle (MEA) et du script de création : ajout d'un référentiel et des compétences.
  • Sankey multi-niveaux (BUT1 → décisions → BUT2 → … → BUT3 → Diplômé / Redoublement / Abandon) + refonte du front-end.
  • Ma contribution : participation à la notation des projets, à la refonte de la base/des requêtes SQL et au diagramme de Sankey.
  • Techniques : Python/Flask, Jinja, SQLite/SQL, modélisation (MEA), Git.
  • Transversales : conduire un projet, optimiser une application, travailler en équipe.
 Démontré : application améliorée à partir d'une base existante (reprise de code).
Exemple : nouveau Sankey multi-niveaux + base restructurée (référentiel, compétences, table SCENARIO).
Justifié : choix de la base appuyé sur une grille de notation /20 (le plus simple à reprendre, pas le mieux noté).		 Finaliser la charte + README ; ajouter des tests automatisés ; déployer l'application.
