Projet n°1 · S3Mnémosyne — Suivi de cohortes (groupe)
ObjectifsTâches individuellesCompétencesPreuvesPistes d'amélioration
Principal : construire une application web de suivi du devenir des cohortes (diagramme de Sankey à partir de l'API ScoDoc).

Secondaire : rendre les parcours lisibles, dans le respect du RGPD (données anonymisées).
  • Principales : j'ai conçu et mis en place l'architecture MVC en Flask (controllers / model / service) — le socle back-end de l'application.
  • Secondaires : développement des routes et de la logique serveur.
  • En équipe : base SQLite, récupération via l'API ScoDoc, première version du diagramme de Sankey.
  • Techniques : Python/Flask, architecture MVC, back-end.
  • Transversales : conduire un projet, collaborer (Git).
 Démontré : architecture Flask fonctionnelle, que j'ai conçue.
Exemple : la structure controllers / model / service a été reprise telle quelle au S4.
Justifié : c'est précisément cette base, que je maîtrisais, qui a été choisie au S4 pour sa facilité de reprise.		 Refondre la base (référentiel + compétences au lieu d'un champ TEXT) ; Sankey multi-niveaux et lisible ; ajouter README + charte ; alléger le front-end.
Projet n°2 · S4Mnémosyne — amélioration (équipe SATURNE 2)
ObjectifsTâches individuellesCompétencesPreuvesPistes (BUT3)
Principal : reprendre une application Mnémosyne du S3 pour refondre sa base de données et enrichir le diagramme de Sankey (parcours plus lisibles).

Secondaire : moderniser l'interface (charte, page d'accueil allégée).
  • Principales : j'ai mené la refonte de la base de données — nouveau modèle entité-association (MEA), script de création et requêtes SQL, ajout d'un référentiel et des compétences (à la place d'un champ TEXT) et d'une table SCENARIO pour les cas de passage.
  • Secondaires : j'ai réalisé l'analyse et la notation des 4 projets S3 (grille langage / BDD / visuel / reprise) qui a guidé le choix de la base.
  • En équipe : diagramme de Sankey multi-niveaux et refonte du front-end / charte.
  • Techniques : modélisation (MEA), SQLite/SQL, Python/Flask.
  • Transversales : optimiser une application, analyser/diagnostiquer, conduire un projet, travailler en équipe.
 Démontré : base de données restructurée que j'ai conçue (référentiel, compétences, table SCENARIO).
Exemple : passage d'un simple champ TEXT à un véritable modèle relationnel.
Justifié : le choix de la base de départ s'appuie sur ma grille de notation /20 (le plus simple à reprendre, pas le mieux noté).		 Ajouter des tests automatisés sur la couche données ; finaliser le README ; déployer l'application.
